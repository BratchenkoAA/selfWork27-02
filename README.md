## 3. Задание 2: Создание схемы и таблиц

### 3.1. SQL-код создания структуры

```sql
-- Создание схемы
CREATE SCHEMA IF NOT EXISTS project_management;

-- Переключение на схему
SET search_path TO project_management;

-- Таблица сотрудников
CREATE TABLE employees (
    employee_id SERIAL PRIMARY KEY,
    first_name VARCHAR(50) NOT NULL,
    last_name VARCHAR(50) NOT NULL,
    email VARCHAR(100) UNIQUE,
    role VARCHAR(50)
);

-- Таблица проектов
CREATE TABLE projects (
    project_id SERIAL PRIMARY KEY,
    project_name VARCHAR(100) NOT NULL,
    start_date DATE,
    end_date DATE,
    status VARCHAR(20) DEFAULT 'Планируется'
);

-- Таблица задач
CREATE TABLE tasks (
    task_id SERIAL PRIMARY KEY,
    project_id INT REFERENCES projects(project_id) ON DELETE CASCADE,
    title VARCHAR(150) NOT NULL,
    due_date DATE,
    priority INT
);

-- Таблица назначений
CREATE TABLE assignments (
    assignment_id SERIAL PRIMARY KEY,
    task_id INT REFERENCES tasks(task_id) ON DELETE CASCADE,
    employee_id INT REFERENCES employees(employee_id) ON DELETE CASCADE,
    assigned_date DATE DEFAULT CURRENT_DATE
);
```

### 3.2. Результат

Созданы 4 таблицы в схеме `project_management`:
- `employees` — информация о сотрудниках
- `projects` — информация о проектах
- `tasks` — список задач
- `assignments` — назначения задач сотрудникам

### 4.2. Распределение задач по проектам

```sql
SELECT 
    p.project_name,
    COUNT(t.task_id) AS количество_задач,
    ROUND(COUNT(t.task_id) * 100.0 / (SELECT COUNT(*) FROM tasks), 1) AS процент
FROM projects p
LEFT JOIN tasks t ON p.project_id = t.project_id
GROUP BY p.project_name
ORDER BY количество_задач DESC;
```

## 6. Задание 5: JOIN запрос

```sql
SET search_path TO project_management;

SELECT 
    t.title AS Название_задачи,
    p.project_name AS Проект,
    e.first_name || ' ' || e.last_name AS Исполнитель,
    t.due_date AS Срок_выполнения,
    t.priority AS Приоритет
FROM tasks t
JOIN projects p ON t.project_id = p.project_id
JOIN assignments a ON t.task_id = a.task_id
JOIN employees e ON a.employee_id = e.employee_id
ORDER BY t.priority, t.due_date;
```

**Результат:** Получена сводная таблица, содержащая:
- Название задачи
- Название проекта
- ФИО исполнителя
- Срок выполнения
- Приоритет задачи

**Пример результата:**
